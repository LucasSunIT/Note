# 分层与模块架构

## 技术栈

- **UI**：Qt 5/6（Widgets、SerialPort、WebSockets、OpenGL 等）
- **语言**：C++17；离线分析通过 **CppBridge** 嵌入 Python（MNE、NumPy）
- **构建**：CMake 单体可执行文件 + 子目录 `CppBridge`、`EegPlotKit`、`Survey`、`Rehab`（可选 `NEUROSTUDIO_ENABLE_*`）

## 逻辑分层（依赖自上而下）

```text
┌─────────────────────────────────────────────────────────┐
│  Views          UI、Presenter、菜单/工具栏工厂            │
├─────────────────────────────────────────────────────────┤
│  Platform       插件接口、IMainShell、IWorkspaceMode      │
├─────────────────────────────────────────────────────────┤
│  Interactor     Controller、Services、Session 编排        │
├─────────────────────────────────────────────────────────┤
│  Core           AppEventBus、Constants                  │
├─────────────────────────────────────────────────────────┤
│  Entities       Study/Subject/Session、EegData、Pipeline  │
├─────────────────────────────────────────────────────────┤
│  Hardware       设备驱动、采集会话、转发                  │
│  Settings       运行时配置（不依赖 EEGDevice）            │
├─────────────────────────────────────────────────────────┤
│  Storage        BDF 录制、RecordingController           │
│  Trigger        外部触发汇入（转发、任务 WebSocket）       │
│  CppBridge      Python/MNE 桥                           │
│  EegPlotKit     时域波形动态库（QCustomPlot）             │
└─────────────────────────────────────────────────────────┘
```

**依赖原则**

- UI 不直接写 Session 指针；写入经 **`DataManagerController`**。
- 硬件访问经 **`hardwareEeg()`** / **`SettingsFacade`**，不新增 `Device/` 实现（legacy 仅转发头）。
- 跨模块广播用 **`AppEventBus`**，避免 MainWindow 与插件硬耦合。

## 主要目录与职责

| 目录 | 职责 |
|------|------|
| `Views/` | 主壳部件：`DataWidget`、`ConfigWidget`、`ContentWidget`；`DeviceDlg` 电极/阻抗 UI；`PlotDlg` 分析图 |
| `Interactor/` | `DataManagerController`、`DataRepository`、导入/备份/报告等 Services；`EEGProcessingManager` |
| `Core/` | `AppEventBus` |
| `Entities/` | 数据模型：`Session` 持有 `EegData`、`PipelineEngine` |
| `Hardware/` | `EEGDevice` 串口协议、`EegDeviceCoordinator` 门面、`EegAcquisitionSession` |
| `Settings/` | `SettingsFacade` 聚合放大器/滤波/电极/存储/转发等配置 |
| `Platform/` | `PluginManager`、`IApplicationPlugin`、`AppContext` |
| `Survey/` | 量表、范式、任务下发（WebSocket）、平板音频 |
| `Rehab/` | 康复设备/方案、Python 运动子进程 |
| `Trigger/` | `TriggerIngressService` 统一触发入口 |
| `Storage/` | BDF 写入、`RecordingController` |
| `Common/` | 配置、PDF、日志工具等 |
| `Device/` | **Legacy**：仅 `#include` 转发，禁止新代码 |

## 构建与 include 根

`NeuroStudioHost` INTERFACE 库将源码根、`Common`、`Core`、`Entities`、`Interactor`、`Views`、`Survey`、`Rehab`、`Hardware`、`Settings` 等加入 include 路径，主目标 `NeuroStudioRehab` 在根 `CMakeLists.txt` 列出大量源文件（物理上仍为单体应用）。

## 启动顺序（概要）

1. `main.cpp`：Python 解释器（CppBridge）、样式、可选登录、`MainWindow`
2. `MainWindow`：`setupWorkspace` → `initApplicationArchitecture`
3. `HardwareContext` / `hardwareEeg()->startMonitoring()` 启动端口扫描监控
4. `registerBuiltInApplicationPlugins()` → Survey/Rehab 插件 `initialize`

详见 [02-主界面与工作区协调.md](./02-主界面与工作区协调.md)。
