# 主界面与工作区协调

## MainWindow 布局

```text
┌──────────────┬────────────────────────────────────────────┐
│ 数据管理 Dock │  中间：ContentWidget（波形/历史 Tab）        │
│ DataWidget   │  + 右侧/底部：日志 Dock                       │
├──────────────┤                                              │
│ 数据处理 Dock │                                              │
│ ConfigWidget │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

- **左**：Study → Subject → Session 树（`DataManagerPresenter` / `DataTreePresenter`）
- **中**：`ContentWidget` — 多 Session Tab、中间 stacked（**历史处理页 / 实时采集页**）
- **左下/可显隐**：`ConfigWidget` — Tab：**数据采集**、**数据处理**、插件注册的 Tab

`initApplicationArchitecture()` 核心步骤：

1. `AppEventBus::wireController(DataManagerController::instance())`
2. 创建 **`EEGWorkspaceCoordinator`**（实现 **`IWorkspaceMode`**），`wire()` 连接 Config/Content/Bus
3. 组装 **`AppContext`**，注册并初始化 **Survey/Rehab** 插件 Tab

## 两套「模式」概念（勿混淆）

| 概念 | 类型 | 谁更新 | 作用 |
|------|------|--------|------|
| **中间 stacked 页** | 0=历史 / 1=实时 | **Config 当前 Tab** → `syncMiddleStackFromConfigTab()` | 决定 Content 显示历史波形还是实时波形 |
| **WorkspaceMode** | Idle / Acquisition / DataProcess | **AppEventBus** → Coordinator `applyMode()` | 业务意图；插件订阅 `workspaceModeChanged`；**不**直接切 stacked |

规则：**数据采集 Tab** → 实时页；**数据处理 Tab** → 历史页。唯一同步入口：`EEGWorkspaceCoordinator::syncMiddleStackFromConfigTab()`。

## EEGWorkspaceCoordinator 连接关系

**Config ↔ Content（局部 UI 协调）**

- Content Tab 切换 → `activateProcessingSession` + `ConfigWidget::updateConfData`
- 处理步骤 `applyStep` / ICA 更新 → `ContentWidget::updateEEGSignalViewer`
- 采集 Tab 离开 → `stopRealCollect`（停 UI 刷新，不停硬件采集）
- 采集开始/停止信号 → `startRealCollect` / `stopRealCollect`
- 阻抗超阈 → 更新通道选择

**AppEventBus（应用级生命周期）**

| Bus 信号 | Coordinator / UI |
|----------|------------------|
| `dataSetOpened` | Config `updateConfData`、Content `openDataSet`、`applyMode(DataProcess)` |
| `dataSetClosed` / `dataSetRenamed` | Content 关 Tab / 改名 |
| `sessionEnterAcquisition` | `applyMode(Acquisition)` |
| `sessionEnterDataProcess` | `applyMode(DataProcess)` |

MainWindow 另订阅 `sessionEnterAcquisition` / `sessionEnterDataProcess`：显示处理 Dock、切换 Config Tab 等（与 stacked 同步通过 `tabChanged` 间接完成）。

## Presenter 与 Controller（数据树命令）

用户操作（打开数据、进入采集）典型路径：

```text
DataManagerPresenter
    → DataManagerController（唯一 Session 状态写入）
        → emit dataSetOpened / acquisitionEntered / ...
            → AppEventBus
                → ConfigWidget / ContentWidget / Coordinator / MainWindow / 插件
```

数据树 CRUD **不经过** Coordinator。

## IWorkspaceMode 与插件

插件通过 `AppContext::workspaceMode` 查询 `workspaceMode()` 并连接 `workspaceModeChanged`。例如 Survey 在采集模式下启停平板音频，与 stacked 页独立。
