# 离线处理与可视化

## EegData 与 Pipeline

每个 **`Session`** 持有 **`EegData`**（`shared_ptr` 工作区）：

- 多 **`EEGDataItem`** 表示 raw、滤波后、epoch、ICA 等阶段
- **`PipelineEngine` / `PipelineManager`**：步骤链（参考、滤波、伪迹、重采样、Epochs、Evoked、ICA…）
- 配置实体在 `Entities/*Conf.h`，UI 在 `Views/ConfigDlg/*Widget`

用户点击处理步骤应用时：

```text
ConfigWidget（DataProcessWidget 等）
    → emit applyStep / applyRemove / …
EEGWorkspaceCoordinator 已 connect
    → ContentWidget::updateEEGSignalViewer
EEGProcessingManager / PipelineEngine
    → 更新 EegData 当前 working item
    → ICA 等 emit icaWaveformNeedsUpdate → 再次刷新 Viewer
```

## CppBridge（Python / MNE）

- 动态库 **`CppBridge`**：嵌入式 Python、`readEegRaw`、MNE 对象与 `QVector` 互转
- **`EegData`** 内可持有 PyObject 相关逻辑（见 `CppBridge/PyObjectUtils.h`）
- 构建依赖本机 Python + NumPy + MNE（CMake 检查 `import mne`）
- 运行时依赖程序目录下 `python/` 嵌入式环境与 `python_scripts/`

离线导入/分析脚本路径与主程序通过 CppBridge API 协作，而非 UI 直接调 Python。

## EegPlotKit

- 独立 CMake 目标 **`EegPlotKit.dll`**
- 对外：`EegPlotHistoryWidget`、`EegPlotRealtimeWidget`、`EEGEvent`、`EegPlotTypes`
- 主程序 `Views/ContentDlg` 包装实时/历史显示；坏道列表由调用方 `setBadChannels()` 传入

## PlotManager 与分析对话框

- `Views/PlotManager`：时域/频域/地形图/诱发电位等对话框入口
- `Interactor/VisualizationManager`：与 Python 可视化样式配置（`main.cpp` 中字体、colorbar 等）
- 大量对话框在 `Views/PlotDlg/`

## Interactor Services（与数据文件）

| Service | 作用 |
|---------|------|
| `DataImportService` | 导入到 Study 目录 |
| `BackupService` | 备份 |
| `EegReportService` | 报告生成 |
| `ConfigExportService` | 配置导出 |

数据树操作核心仍在 **`DataManagerController`** + **`DataRepository`**。

## Storage

- **`BdfFileWriter`** / **`BdfFileValidator`**：BDF 读写校验
- 实时录制由 **`RecordingController`** 协调，配置来自 Settings
