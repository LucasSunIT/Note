# 实时采集与硬件数据流

## 入口 API

| API | 返回/作用 |
|-----|-----------|
| `HardwareContext::instance()` | 初始化注册表 |
| **`hardwareEeg()`** | `EegDeviceCoordinator*` — UI/Interactor **首选** |
| **`hardwareAcquisition()`** | `IAcquisitionDevice*` — 仅连接 + `startStream`/`stopStream` 契约 |
| **`SettingsFacade::instance()`** | 放大器/电极/双通道/滤波 UI/存储路径/转发端口等 |

`EegDeviceCoordinator` **继承** `IAcquisitionDevice`，并组合：

- `EegConnectionService` — 串口扫描、连接、`EEGDevice` 生命周期
- `EegAcquisitionSession` — collecting 状态、Session 绑定、`EEGProcessor`
- `EegForwardController` — TCP 数据转发

## IAcquisitionDevice vs 业务采集

| 层级 | 方法 | 含义 |
|------|------|------|
| 契约层 | `startStream()` / `stopStream()` | 设备侧开始/停止 ADS1299 数据流 |
| 业务层 | `startCollectingSession(Session*)` | 绑 Session、detectChannelMode、置 collecting，**内部**再 `startStream()` |
| 业务层 | `stopCollectingSession(reason)` | 停录、停流、清理采集会话 |

**UI「开始采集」必须走 `startCollectingSession`**，不要对界面直接 `startStream()`（除非底层联调）。

## 实时数据流（简图）

```text
串口 EEGDevice
    │ 原始帧 (EEGFrame)
    ▼
DualChannelBuffer 合并（双放大器）
    ▼
EEGProcessor（QThread）
    │ 写入
    ▼
EegData（当前 Session 的内存缓冲）
    │ 信号/轮询
    ├── RealTimeEEGSignalViewer / EegPlotRealtimeWidget（显示）
    ├── RecordingController → BdfFileWriter（磁盘）
    └── EegForwardController → DataForwardService（TCP 转发）
```

`EEGProcessor::pushChannelFrame` 由设备线程侧喂帧；合并配置来自 `SettingsFacade::dualChannel()`。

## 配置 vs 硬件（Settings 边界）

- **`AmplifierConfig`**：设备意图（采样率标签、增益等），下发 `EEGDevice` / BDF 头
- **`FilterConfig`**：软件实时滤波 Hz，供 `RealTimeEEGProcessor`、RtFilterWidget
- 改寄存器/串口 → `amplifier()` / `dualChannel()`；改屏幕滤波 → `filter()`

## 触发（Trigger）汇入

`TriggerIngressService::submit(TriggerEvent)`：

- **DataForward**：转发包中的 event code → `hardwareEeg()->injectTriggerCode`
- **TaskWebSocket**：Survey 任务平板时间戳 → 映射为实时 Annotation（`realtimeAnnotationAdded`）

采集停止时清空平板 timing anchor。

## 录制

- `EegDeviceCoordinator::startRecording` / `stopRecording` → `RecordingController`
- 路径与通道信息来自 `SettingsFacade::storage()`、`electrode()` 等
- Survey 下发任务前可检查 `hardwareEeg()->isRecording()`

## 启动与退出

- `main`：`hardwareEeg()->startMonitoring()` 启动端口扫描
- `MainWindow::closeEvent`：若 `isCollecting()`，确认后 `stopCollectingSession`

## Legacy Device/

`Device/` 仅兼容 `#include` 转发；实现均在 `Hardware/` + `Settings/`。详见仓库 `docs/HARDWARE_DEVICE_CUTOVER.md`。
